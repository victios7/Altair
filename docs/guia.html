1. Visión general

Altair es un lenguaje compilado estáticamente y orientado a expresiones que transpila a C mediante altairc. Cada programa .at se convierte en un binario nativo sin dependencias en tiempo de ejecución.

CaracterísticaDescripción
Storage tiersCada variable declara dónde vive: ram, disk, cache o temp
Servidor HTTPlisten y route son palabras clave del lenguaje, sin frameworks externos
OrbitVariables que migran entre tiers en tiempo de ejecución
Gráficos v1.6.5vCVentana y loop de juego vía Raylib (link * raylib)
Binario nativoCompila a un ejecutable sin runtime instalado
PlataformasWindows (10/11), Linux, macOS

2. Pipeline del compilador

lexer.c
→
parser.c
recursivo descendente
→
sema.c
análisis semántico
→
codegen.c
genera C
→
gcc -O2
binario nativo

El compilador escribe el C generado a un temporal /tmp/altair_<pid>_gen.c, invoca gcc -O2 y lo elimina. En Windows usa el mingw64 incluido en el instalador.

/ Ver el C generado sin compilar:
altairc programa.at --emit-c

/ Ver el número de nodos del AST:
altairc programa.at --emit-ast

3. Estructura del programa

altair.doc;
    name = "MiApp"
    version = "1.0"
    author = "Tu Nombre"
create altair.doc

/ Esto es un comentario

log "Hola desde Altair!"

La cabecera (altair.doc; … create altair.doc) es opcional. Campos disponibles: name, version, author.

Comentarios: una / al inicio del token (o tras espacio) seguida de espacio o letra inicia un comentario de línea. Si el token anterior era un valor, la / se interpreta como división.

4. Variables y almacenamiento

Declaración

define <tipo> <nombre> [storage] [calificadores] [= expresión]

La palabra clave define es opcional:

/ Con define:
define numeric contador = 0 ram

/ Sin define (equivalente):
numeric contador = 0 ram
text mensaje = "Hola" ram
bool activo = true ram

Tiers de almacenamiento

KeywordComportamiento
ramMemoria del proceso, bloqueada con mlock. Rápida; se pierde al salir.
diskPersiste en ~/.altair/<app>/disk/<var>.altv. Sobrevive reinicios.
cachePersiste con TTL opcional. Se auto-expira cuando expira el TTL.
tempSe pone a cero en release. Para datos sensibles.
autoEl runtime elige el tier (equivale a ram por defecto).
define text usuario disk                      / persiste entre ejecuciones
define text token_sesion temp                 / se borra al liberar
define text respuesta cache expire=30m        / expira en 30 minutos

Calificadores

CalificadorEjemploEfecto
constdefine numeric PI const = 3.14159No se puede reasignar (lanza ALT0007)
expire=<dur>cache expire=5mAuto-expira tras la duración
weight=<n>weight=5Hint de prioridad (entero ≥ 0)

Duraciones: 30s (segundos), 5m (minutos), 2h (horas).

Asignación

nombre = "Alicia"
contador += 10
contador -= 3
contador *= 2
contador /= 4
contador %= 3
lista[0] = "nuevo"

7. Control de flujo

Todos los bloques se cierran con break. La cadena if/elif/else comparte un único break final.

if / elif / else

if puntuacion > 90;
    log "Excelente"
elif puntuacion > 70;
    log "Bien"
else;
    log "Necesita mejorar"
break

while

numeric n = 1
while n <= 10;
    log n
    n += 1
break

repeat

repeat 5 times;
    log "tick"
break

/ La palabra 'times' es opcional:
repeat contador;
    log "iteracion"
break

forever

forever;
    / bucle infinito
    if condicion_salida;
        exit
    break
break

foreach

list colores = ["rojo", "verde", "azul"]
foreach color in colores;
    log color
break

La variable iteradora se crea automáticamente y se libera al salir del bucle.

exit / wait

/ Dentro de un bucle: rompe el bucle
/ Fuera de un bucle: termina el programa
exit

/ Pausa la ejecución
wait 2s
wait 500    / sin sufijo = segundos

8. Funciones

/ Función sin retorno:
fun saludar text nombre;
    log "Hola, " + nombre
break

/ Función con retorno (tipo antes de los parámetros):
fun sumar -> numeric numeric a, numeric b;
    return a + b
break

/ Función recursiva:
fun factorial -> numeric numeric n;
    if n <= 1;
        return 1
    break
    return n * factorial(n - 1)
break

Llamada

saludar("Alicia")
numeric resultado = sumar(3, 4)

/ Sin paréntesis (un argumento):
saludar "Alicia"
Las variables globales son accesibles y modificables desde dentro de las funciones. El codegen las busca vía altair_var_lookup si no están declaradas localmente.

9. Clases y objetos

class Jugador;
    text nombre = "" disk
    numeric puntos = 0 ram
    bool activo = true ram

    fun sumarPuntos numeric cantidad;
        puntos += cantidad
    break

    fun obtenerPuntos -> numeric;
        return puntos
    break
create class

/ Instanciar:
object jugador = Jugador() ram

/ Usar:
jugador.nombre = "Alicia"
jugador.sumarPuntos(10)
numeric pts = jugador.obtenerPuntos()
log jugador.nombre + " tiene " + pts + " puntos"

10. Listas

list items = ["a", "b", "c"]

log items[0]          / "a"
items[1] = "X"        / asignación por índice

items.append("d")     / añade al final
items.remove(0)       / elimina por índice (devuelve bool)
items.clear()         / vacía la lista
log items.length      / número de elementos (sin () también funciona)

/ Concatenar dos listas:
list total = lista1 + lista2

/ Iterar:
foreach item in items;
    log item
break

Los índices fuera de rango lanzan ALT0013.

11. Manejo de errores

try;
    numeric resultado = 10 / 0
break
catch as err;
    / Variables disponibles: err_code, err_message, err_line
    log "Error " + err_code + ": " + err_message
    log "Línea: " + err_line
break

Los bloques try/catch pueden anidarse hasta 128 niveles.

12. Snapshots

Un snapshot serializa todas las variables disk y cache activas a un archivo en ~/.altair/<app>/snap/.

snapshot create "partida_1"
snapshot restore "partida_1"
snapshot delete "partida_1"

13. Choose (aleatorio ponderado)

Selecciona un valor aleatoriamente según pesos porcentuales. Deben sumar 100.

choose resultado;
    50% = opcionA
    30% = opcionB
    20% = opcionC
define

log resultado    / tipo text: nombre de la opción elegida

14. Introspección

/ Sistema:
log system@os          / "windows" | "linux" | "macos"
log system@memory      / memoria disponible en bytes
log system@cpu         / número de CPUs

/ Tier actual de una variable:
log system@locate(miVariable)   / "ram" | "disk" | "cache" | "temp"

/ Compilador:
log compiler@version   / "1.6.5vC"
log compiler@name      / "altairc"

/ Programa (valores de altair.doc):
log program@name
log program@version
log program@author

15. Token (valor de un solo uso)

Un token envuelve un valor y solo puede leerse una vez. Tras la primera lectura queda marcado como consumido.

define token secreto = "clave-efimera" temp

log secreto    / devuelve "clave-efimera" y lo marca consumed
log secreto    / devuelve [token:consumed]

16. Orbit y Prefer

Orbit

Define una variable con varios estados de almacenamiento, cada uno mapeado a un tier diferente.

numeric sessionScore = 0 orbit;
    1 = create temp      / estado 1: temporal (inicio)
    2 = active ram       / estado 2: en memoria (activo)
    3 = inactive cache   / estado 3: en caché (pausa)
    4 = finish disk      / estado 4: en disco (fin)
break

/ Migrar por número:
sessionScore migrate 2

/ Migrar por nombre:
sessionScore migrate active

Los números de estado deben ser únicos (ALT0009). La variable comienza en el primer estado declarado.

Prefer

Define un orden de preferencia de tiers. El runtime elige el primero disponible.

define text datos prefer ram, cache, disk;

17. Release

Libera explícitamente la memoria de una variable. Para temp, pone a cero la memoria antes de liberar.

define numeric temporal = 42 ram
/ ... uso ...
release temporal    / seguro aunque ya se haya liberado antes

18. Entrada del usuario

/ Con prompt y tipo:
text nombre = user input "¿Cómo te llamas?" as text
numeric edad = user input "¿Cuántos años tienes?" as numeric

/ Sin tipo (inferido como text):
text entrada = user input "¿Algo más?"

/ Sin prompt:
text resp = input as text

19. Servidor HTTP v1.6.5vB+

El servidor HTTP está implementado en el runtime de Altair usando sockets POSIX directamente (sin frameworks externos). Es de un solo hilo; las conexiones se atienden secuencialmente.

listen 8080;
    / rutas y configuración aquí
break

/ O con variable:
numeric PUERTO = 3000 ram
listen PUERTO;
    / ...
break

20. Rutas y handlers

listen 8080;
    route "GET" "/hola";
        respond.json("{\"msg\":\"hola\"}")
    break

    route "POST" "/datos";
        text cuerpo = body()
        text usuario = header("X-User")
        text id = param("id")
        respond.status(201)
        respond.json(cuerpo)
    break
break

Métodos disponibles en el handler

FunciónDescripción
body()Cuerpo crudo de la petición (text)
header("nombre")Valor de una cabecera HTTP
param("nombre")Parámetro de query string
respond.json(expr)Responde con Content-Type: application/json
respond.text(expr)Responde con Content-Type: text/plain
respond.status(n)Establece el código de estado HTTP
stopDetiene el procesamiento (útil en middleware)

21. Middleware

Se ejecuta antes de cada ruta. Si llama a stop, la petición no llega al handler.

middleware autenticacion;
    text clave = header("X-API-Key")
    if clave != "mi-secreto";
        respond.status(401)
        respond.json("{\"error\":\"no autorizado\"}")
        stop
    break
break

22. Rate limiting

route "POST" "/api" rate_limit 60 per_minute;
    respond.json("ok")
break

route "GET" "/buscar" rate_limit 10 per_second;
    respond.json("resultados")
break

Si se supera el límite, el runtime devuelve automáticamente 429 Too Many Requests.

23. Health checks

health "/health";
    check "base_datos" -> 1      / 1 = OK, 0 = fallo
    check "almacenamiento" -> 1
break

Devuelve JSON con el estado de cada check y un status global (ok o degraded).

24. Métricas

metrics "/metrics";

Expone un endpoint compatible con Prometheus con métricas del runtime (peticiones totales, errores, tiempos de respuesta).

25. Graceful shutdown

on_shutdown;
    log "Servidor deteniendo..."
    / limpiar recursos, cerrar conexiones...
break

26. Sesiones

session usuario_sesion expires 30m;

Las sesiones se almacenan en memoria del runtime, indexadas por ID de sesión que el cliente proporciona.

27. Config y variables de entorno

config;
    env("PORT") default "8080"
    env("API_SECRET") required
    env("LOG_LEVEL") default "info"
break
  • default "valor": usa ese valor si la variable no está definida.
  • required: el programa falla al arrancar si la variable no está definida.

28. Pool de base de datos

db_pool conexion = connect("postgresql://usuario:pass@host:5432/db") max 10

29. Jobs programados

Funciones que se ejecutan periódicamente mientras el servidor está en marcha. Se ejecutan en el hilo principal entre peticiones.

job limpieza every 5m;
    log "Limpieza ejecutada"
break

job heartbeat every 30s;
    log "Servidor activo"
break

30. Gráficos y UI con Raylib v1.6.5vC

La versión 1.6.5vC añade soporte de gráficos 2D y UI mediante Raylib. El compilador detecta automáticamente el uso de link * raylib y enlaza la librería.

Activar y estructura básica

link * raylib

window;
    title = "Mi Juego"
    width = 800
    height = 600
    fps = 60
create window

loop;
    clear black

    draw text;
        text = "Hola desde Altair!"
        x = 100
        y = 100
        size = 24
        color = white
    create draw

    if key "SPACE";
        log "Espacio pulsado"
    break
break

Primitivas de dibujo

draw rectangle;
    x = 50
    y = 50
    width = 200
    height = 100
    color = red
create draw

draw circle;
    x = 400
    y = 300
    radius = 50
    color = blue
create draw

draw line;
    x1 = 0
    y1 = 0
    x2 = 800
    y2 = 600
    color = green
create draw

draw image;
    image = miImagen
    x = 100
    y = 100
create draw

Imágenes

image fondo = "fondo.png" disk
image sprite = "personaje.png" ram

Audio

sound disparo = "disparo.wav"
music musica_fondo = "musica.mp3"

play musica_fondo
pause musica_fondo
stop musica_fondo
play disparo

Teclado

/ Teclas: SPACE, ENTER, ESCAPE, A-Z, 0-9, F1-F12, UP, DOWN, LEFT, RIGHT...
if key "LEFT";
    jugador_x -= velocidad
break
if key "RIGHT";
    jugador_x += velocidad
break

Colores predefinidos

white, black, red, green, blue, yellow, orange, purple, pink, gray, lightgray, darkgray, brown, skyblue, darkblue, maroon, darkgreen, lime, gold, beige, magenta, violet, darkpurple, darkbrown, raywhite, transparent.

/ Color personalizado:
color miColor = rgb(255, 128, 0)
color miColorHex = "#FF8000"

Escenas

scene menu;
scene juego;
scene fin;

goto menu
goto juego

31. CLI de altairc

Altair Compiler v1.6.5vC

Uso:
  altairc <archivo.at> [opciones]
  altairc guide              Escribe ALTAIR_GUIDE.md en el directorio actual
  altairc guide --stdout     Imprime la guía en stdout

Opciones:
  -o <salida>       Nombre del binario de salida (por defecto: a.out / a.exe)
  -icon <file.ico>  Incrusta un icono .ico en el .exe (solo Windows)
  --emit-c          Imprime el C generado en stdout (no compila)
  --emit-ast        Imprime el número de nodos del AST
  --no-sema         Omite el análisis semántico
  -v, --version     Muestra la versión del compilador
  -h, --help        Muestra esta ayuda

Ejemplos de uso

/ Compilar y ejecutar (Linux/macOS):
altairc programa.at -o programa
./programa

/ Compilar y ejecutar (Windows):
altairc programa.at -o programa.exe
programa.exe

/ Depurar el C generado:
altairc programa.at --emit-c | head -100

/ Generar la guía en el directorio actual:
altairc guide

/ Instalar (desde el fuente, Linux/macOS):
make
make install    / instala en ~/.local/bin/altairc
